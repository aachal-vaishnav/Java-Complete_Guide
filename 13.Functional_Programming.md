## **13.1 What is Functional Programming?**  
ðŸ“Œ **Functional programming focuses on writing concise, declarative functions instead of step-by-step instructions.**  

âœ… **Core Principles:**  
âœ” **First-Class Functions** â†’ Functions are treated like objects (passed as arguments, returned as values).  
âœ” **Immutable Data** â†’ Data should not be modified after creation.  
âœ” **Pure Functions** â†’ Functions produce the same output for the same input (no side effects).  
âœ” **Higher-Order Functions** â†’ Functions take or return other functions.  

âœ… **Example Using Functional Programming:**  

ðŸ“Œ **Streams process data efficiently without storing them in memory.**  

âœ… **Characteristics:**  
âœ” **No Storage** â†’ Operates directly on data sources like arrays, lists.  
âœ” **Functional Operations** â†’ Supports transformations like `map()`, `filter()`.  
âœ” **Lazy Execution** â†’ Processes data **only when needed**.  
âœ” **Parallel Processing** â†’ Supports multi-threaded processing.  

âœ… **Example Using Streams:**  

## **13.7 Functional vs Structural Programming**  
ðŸ“Œ **Comparison between Functional & Imperative programming styles.**  

| **Feature**              | **Functional Programming** | **Imperative Programming** |
|-------------------------|----------------------|-------------------|
| **Execution**          | Declarative logic    | Step-by-step instructions |
| **Code Complexity**    | Less complex         | More complex |
| **Side Effects**       | Avoided (Pure Functions) | Mutations are allowed |
| **Optimization**       | Easier optimization  | Harder optimization |

âœ… **Example Using Imperative Approach:**  
```java

for (int num : numbers) { sum += num; }
```

âœ… **Example Using Functional Approach:**  
```java

```

âœ… **Terminal Operations (`Execution Trigger`):**

âœ… **Example Using Intermediate & Terminal Operations:**
